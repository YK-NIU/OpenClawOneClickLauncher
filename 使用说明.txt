小河狸养龙虾 启动器 (OpenClaw 本地入口)

它是什么
- 这是一个 Windows 桌面启动器，用来“安装并启动” OpenClaw 的本地 Gateway 与 Dashboard。
- 目标是让同事开箱即用：跳过 OpenClaw 复杂的交互式设置，把关键步骤自动化。

你需要准备什么
- 一个阿里云百炼 (DashScope) API Key。
  在启动器界面里点“创建 API Key”即可打开控制台页面。

如何使用 (推荐流程)
1) 双击运行“小河狸养龙虾.exe”。
2) 展开“API Key 配置”，粘贴你的百炼 API Key，点“保存 API Key”。
3) 点“启动”。
   - 第一次启动：会自动安装必要组件并安装/配置 OpenClaw，然后启动本地 Gateway，并自动打开 Dashboard。
   - 之后再次打开：一般会直接启动，不会重复安装。

它做了哪些事情 (真实行为)
- 依赖检查：如果系统缺少 Node.js(需要 22+) 或 Git，会尝试通过 winget 在线安装/升级。
- 安装 OpenClaw：通过 npm 全局安装 openclaw 到启动器自己的目录下。
- 一键初始化：以“本地 Gateway + 百炼 OpenAI 兼容接口”的方式非交互配置 OpenClaw。
  默认模型顺序(主用 -> 备用)：
  - qwen3.5-plus
  - MiniMax/MiniMax-M2.5
  - deepseek-v3.2
- 启动本地 Gateway：绑定到 127.0.0.1（仅本机可访问），端口 18789。
- 打开 Dashboard：启动器会自动打开本机浏览器。

为什么说“风险较低”
- Gateway 只绑定到本机回环地址(127.0.0.1)，不会对局域网或公网暴露端口。
- 启动器只保存你的 API Key 到本机配置文件中，用于给 OpenClaw 调用模型时使用。
- 重要说明：模型调用仍然会发往你选择的云服务商(默认百炼)。是否有免费额度/试用额度以阿里云实际政策为准。

成功部署后如何切换其他大模型
- 等你确认 OpenClaw 在本机跑通后，再在 OpenClaw 的 Dashboard/配置中修改成你需要的其他模型或服务商即可。
  也就是说：先用默认配置跑通“安装+启动”链路，再做个性化配置。

卸载
- 点击启动器里的“卸载”，会卸载 OpenClaw 并清理启动器自己的安装目录(不强制清理你用户目录下的 OpenClaw 配置)。

常见问题
- 公司网络限制导致安装失败：打开“查看日志”查看原因；必要时在“高级设置”里切换 npm 镜像。
- winget 不可用：需要你手动安装 Node.js 22+ 与 Git，再重新点击“启动”。
